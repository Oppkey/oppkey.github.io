Building a New Software Developer Community for a Hardware Company On a Limited Budget

Problem: Hardware company with a limited budget needed to build a software community around a new API

Solution: Build community-based API usage documentation and developer blog posts. Apply marketing and engagement techniques with disciplined approach to metrics to ignite community activity.

Results:

  • 2,000 developers registered for opt-in email list, built from zero
  • 20,000 monthly pageviews to new developer documentation and use cases
  • 130+ developers registered for Silicon Valley meetup group
  • Sharp increase in US sales


Activities That Delivered Developer Engagement

  • Developer events, selected, planned and staffed
  • Collect email addresses, build growing list
  • Bay Area Maker Faire, TechCrunch Disrupt Hackathon, DeveloperWeek Hackathon
  • Email list management of promotion campaigns
  • Use MailChimp, show email blast analytics
  • Developed and promoted special “Unofficial Guides” for developers
  • API Usage, Media Management (360° images and videos), Blog

In 6 months, you’ll have results that will lead to a serious devrel commitment

Additional Activities

  • Managed GitHub repository of community code examples
  • Forum engagement, approximately 100 posts per month
  • Developed sample applications
  • Python library to access API
  • JavaScript/NodeJS examples
  • Seeded community applications by building examples ourselves to show final product concepts
  • Focused on building applications in key verticals where company had the most sales: real estate, automotive, travel,
  • Hardware prototyping with "hobbyist" or "hardware hacker" types of board computers such as a portable Raspberry Pi and touchscreen for control experiments and ESP8266 WiFi controller for physical button control signalling
  • Ran experiments on four different blog and forum or discussion platforms: phpBB, Discourse, Jekyll on GitHub Pages with Disqus, Wordpress with integrated commenting.
  • Collected detailed analytics on what the community read, what they clicked on, how much time spent reading, where the referral traffic came from, what activities worked.
  • Promoted developer contest
